# 强制规范

## 1. YAML 配置规范

- 必须包含字段：skill_id（下划线小写命名）、skill_name、description、version、author、trigger、params
- trigger 下必须包含：
  - keywords：3-5个触发关键词，覆盖用户常用口语说法
  - semantic_desc：一句话语义描述，用于语义触发
- params 是参数列表，每个参数必须包含：name、type（仅允许 string/int/bool/float）、required（true/false）、default、description
- 严格遵循 YAML 语法：冒号后必须加空格，同层级空格对齐，禁止使用 Tab，字符串特殊字符加引号
- 不要使用复杂嵌套，保持结构扁平易读

## 2. Python 执行代码规范

- 只写一个核心执行函数，函数名与 skill_id 保持一致，入参与 YAML 中 params 一一对应
- 函数顶部写完整文档字符串，说明功能、入参、返回值
- 加入基础异常捕获，出错返回友好的中文提示，不要直接抛异常
- 代码底部必须加 `if __name__ == "__main__":` 单独测试块，填入默认参数，方便用户直接运行测试
- 用到的第三方依赖要精简、常见，不要用冷门库
- 代码加清晰中文注释，关键步骤标注说明，适合小白阅读
