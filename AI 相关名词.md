#### Agent AI智能体

AI Agent = **具备自主规划、思考、调用工具、完成复杂任务的智能体**

单纯 LLM 只会聊天、写文字；Agent 是在 LLM 大脑基础上，加上**思考逻辑、工具能力、记忆、执行循环**，能自己拆解任务、主动操作外部系统，不用人一步步指挥。

核心运行循环（Agent 经典工作流）：

1. 接收用户需求
2. LLM 思考：判断自己能不能直接回答，还是要调用工具
3. 调用 Skill/MCP 工具：查数据库、读文件、运行脚本、查接口
4. 获取真实数据后整合信息，生成最终答案
5. 自我校验，发现信息不足则重复查询，解决幻觉问题

#### FDE       Forward Deployed Engineer，前线 / 前置部署工程师

AI 落地专属复合型岗位,**长期驻场客户内网 / 业务现场**，打通 AI 工具和企业业务的最后一公里。

三大核心工作

1. 需求挖掘：摸清客户真实业务痛点，梳理 AI 落地流程；
2. 现场开发部署：搭建 Agent、RAG、工作流、对接客户内部系统；
3. 双向反馈：现场踩坑、优化方案，把客户真实需求回流给总部产品团队迭代通用 AI 平台。

能力要求

大模型开发、Agent 搭建、数据库、业务理解、客户交付、运维一体。

#### skill（AI Agent 技能）

通俗定义

AI 智能体（Agent）的**可插拔业务能力插件**，一套封装完整执行流程、工具调用、业务规则的标准化任务包。

类比

Agent 是人，Skill 就是人掌握的单项专业技能：查报表、订行程、数据清洗、生成合同、调用数据库查询。

Skill 包含**完整操作步骤、异常处理、校验规则**，AI 能自主按流程跑完完整业务。

#### 蒸馏（知识蒸馏 Knowledge Distillation）

通俗定义

AI 模型压缩 & 知识迁移技术：**大模型当老师，小模型当学生，把大模型的思考逻辑、推理能力提炼给小模型**。

核心原理

不只让小模型学标准答案（硬标签），还要学大模型输出的概率分布、推理思路、类别关联（暗知识），小模型参数量大幅缩小，但效果接近大模型。

========================================================================================================================

#### MCP（Model Context Protocol 模型上下文协议）

作用：统一规范大模型 / Agent 与外部文件、数据库、API、本地工具、业务系统的交互方式，解决各家 Function Calling、插件互不兼容、重复开发适配器的痛点。

底层基于 JSON-RPC 2.0，支持本地进程、HTTP 两种传输通道。

MCP 是 Skill 跨模型复用的统一传输协议；

#### LLM

Large Language Model  **大语言模型**

基于海量文本数据训练出来的巨型人工智能模型，核心能力是**理解、生成人类语言**，能阅读理解、写文案、逻辑推理、代码编写、规划任务。

#### AI 幻觉（Hallucination)

AI 幻觉特指**大语言模型 LLM 凭空生成虚假、不存在、和事实矛盾的内容**，模型本身不知道自己在说谎，只是概率生成文字导致错误输出。



为什么 LLM 会产生幻觉

1. **生成机制是概率预测**
   LLM 只会按文字出现概率逐字续写，没有真实 “知识库记忆”，遇到模糊、未知信息时，会自动补全通顺文字，而非直白说 “我不知道”。

2. **上下文信息不足**
   只给少量资料、缺少参考原文，模型只能靠训练里的通用知识脑补，极易出错。

3. **模型轻量化副作用**
   经过**知识蒸馏**后的小模型，参数量少，知识压缩丢失细节，幻觉概率远高于大教师模型；本地 Ollama 跑 7B 小模型更容易胡说。

4. **Prompt 引导不当**
   强制要求模型必须给出完整答案、禁止说不清楚，会逼迫模型编造内容。
