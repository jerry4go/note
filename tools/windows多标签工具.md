提示词

你是Windows桌面C++/C#开发资深工程师，现在需要开发一款Windows10/Win11文件资源管理器（explorer.exe）多标签增强插件，类似QTTabBar、Clover功能。

请完整输出：

1. 整体技术选型对比（C++ COM插件 / C# WinForms注入 / .NET 外壳扩展），说明各自优缺点、兼容性、崩溃风险
2. Windows资源管理器外壳扩展、Shell Extension、Explorer Band原理讲解
3. 完整项目分层架构设计
4. 核心功能清单（必做+可选拓展）
5. 开发环境、依赖库、编译部署流程
6. 注册表注册、插件加载机制
7. 风险点：explorer进程崩溃、多版本系统兼容、权限问题、资源泄漏解决方案
   
   
   
   核心需求：1. 给原生explorer顶部增加浏览器式多标签栏，Ctrl+T新建标签、Ctrl+W关闭标签2. 每个标签独立路径，支持拖拽标签排序、拖拽文件夹新建标签3. 标签记忆上次打开路径，重启资源管理器自动恢复标签页4. 支持文件夹书签收藏、快捷键自定义5. 兼容Win10 1903~22H2、Win11全版本，32/64位适配6. 轻量低占用，不频繁卡死资源管理器7. 右键文件夹可“在新标签中打开”菜单拓展禁止使用第三方闭源框架，尽量依托Windows原生Shell API，输出内容结构清晰，分模块讲解。
