# 运维智能体灰度上线执行 Checklist

> 适用范围：IT运维自愈智能体（多智能体版）生产灰度落地
> 执行周期：共31天（准备1天 + 影子模式7天 + 小流量自愈10天 + 全量上线14天）
> 核心原则：每项完成并验证通过后，方可进入下一环节；出现异常立即执行回滚

---

## 阶段0：上线前准备日（Day 0）

### 一、生产环境部署检查

- [ ] FastAPI工具服务生产环境部署完成
  - 验证：访问 `http://生产内网IP:8001/docs`，Swagger文档正常展示
  - 验证：所有接口手动测试通过，返回格式与字段完整
- [ ] Dify生产环境5个应用配置完成
  - 验证：告警预处理Agent、调度总管Agent、系统专家、数据库专家、网络专家全部创建完成
  - 验证：所有工具配置、知识库绑定、系统提示词已同步至生产环境
  - 验证：每个Agent的API密钥生成完成，权限隔离正确
- [ ] 生产向量知识库搭建完成
  - 验证：历史故障案例库、运维SOP库、系统架构库3个知识库已导入生产数据
  - 验证：召回测试正常，Top3结果匹配度符合预期
- [ ] 钉钉推送通道配置完成
  - 验证：测试消息可正常到达运维群，格式、@人功能正常
  - 验证：P1/P2/P3分级样式展示正确

### 二、灰度与风控配置校验

- [ ] 告警白名单过滤机制生效
  - 验证：非白名单IP的告警，不进入智能体全链路，直接走原告警通道
  - 验证：白名单内IP的告警，可正常进入预处理流程
- [ ] 三级熔断开关全部测试通过
  - [ ] 自愈总开关：切换为关闭后，所有自愈接口自动降级为试运行模式
  - [ ] 告警转发开关：关闭后，告警不再进入智能体系统
  - [ ] 应用停用开关：停用Dify应用后，API调用自动走兜底流程
  
  - 验证：所有开关切换后10秒内生效，无残留执行
- [ ] 白名单初始状态确认
  - 排障诊断白名单：空（影子模式阶段全量放开）
  - 自动自愈白名单：空（阶段2再逐步添加）
- [ ] 审计日志功能验证
  - 验证：所有工具调用、自愈操作均有完整日志记录
  - 验证：每条记录包含操作人、时间、参数、结果四要素

### 三、基线数据与预案准备

- [ ] 上线前一周基线数据统计存档
  - 平均故障排障时长：____ 分钟
  - 日均人工处理告警数：____ 条
  - 常见故障人工处理准确率：____ %
- [ ] 回滚方案全员周知，紧急联系人清单同步
- [ ] 灰度期间每日站会时间、复盘机制确认

## 附2：紧急回滚操作 Checklist

触发条件：出现误操作、系统异常、业务影响

### 一级回滚（模块级，10秒）

- [ ] 关闭对应功能模块开关
  - 自愈异常：关闭自愈总开关，全局降级为试运行模式
  - 预处理异常：关闭聚合功能，告警直接进入主排障
- [ ] 验证：异常操作已停止
- [ ] 记录故障现象与触发时间

### 二级回滚（执行级，10秒）

- [ ] 一键关闭所有自动执行权限
- [ ] 全局降级为纯建议模式
- [ ] 通知运维全员：智能体仅作参考，所有操作人工执行

### 三级回滚（全局级，30秒）

- [ ] 关闭告警转发开关，告警完全走原有人工通道
- [ ] 停用所有Dify智能体应用
- [ ] 通知全员：智能体系统下线，恢复纯人工处理
- [ ] 启动故障排查，定位问题根因
