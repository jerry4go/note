* 温度 = 0.1：学霸死记硬背标准答案，一字不差，绝对不发挥，肯定对，但很死板
* 温度 = 0.7：普通好学生，按标准答案为主，偶尔换个说法，既靠谱又灵活
* 温度 = 1.8：学渣放飞自我，想到啥写啥，偶尔有金句，但大部分时候不靠
  
  

* Temperature：管 “高低概率的词，选中的差距大不大”—— 数值高了，冷门词也有很大机会被选中

 3 个必学的进阶小技巧

1. **思维链技巧**：做数学题、逻辑题、复杂任务，结尾加一句「请一步步思考，写出推理过程再给答案」，准确率直接提升。
2. **防幻觉技巧**：加一句「如果不知道答案，直接说我不知道，不要编造内容」，减少 AI 胡说八道。
3. **迭代优化技巧**：一次不满意，不要重写整个 Prompt，直接告诉它「刚才的回答太笼统了，把第二部分再细化一下，加 2 个具体案例」，改起来更快

### LlamaIndex 是什么

大白话：**专门用来做 RAG 的「搭积木」工具包（框架）**。



* **LangChain**：把大模型、RAG、工具、记忆等能力封装成标准化 “积木”，拼一拼就能做出完整 AI 应用，不用从零造轮子
* **LangGraph**：专门编排复杂工作流，支持条件分支、循环执行、人工介入，是开发 AI 智能体（Agent）的主流工具
  
  

Chain（链）是 LangChain 的核心设计思想：把多个组件按固定顺序串联起来，前一步的输出自动作为后一步的输入，全程自动执行，不用手动一步步调用。



LangGraph 是 LangChain 官方推出的**工作流编排框架**，专门用于搭建有循环、有分支、有状态的复杂 AI 流程，是目前开发 AI 智能体（Agent）的主流方案。



### LangGraph 四大核心要素

1. **状态（State）**：整个流程的共享数据区，所有节点都可以读取、修改里面的数据
2. **节点（Node）**：一个个具体执行步骤，比如 “检索资料”“生成回答”“审核内容”，每个节点对应一个执行函数
3. **边（Edge）**：连接节点的箭头，代表执行的先后顺序
4. **条件边（Conditional Edge）**：带判断逻辑的箭头，根据状态中的数据，决定下一步流向哪个节点

### Transformer 到底是什么

大白话定义：**Transformer 是 2017 年提出的一种 AI 模型架构，核心是「自注意力机制」，它彻底解决了之前 AI 模型 “记不住长文本、训练速度慢” 的问题，是现在所有大语言模型、AI 绘画、语音模型的底层基础。**



自注意力的作用：**模型处理每个词的时候，都会自动计算这句话里所有词和它的关联程度，重点关注相关性高的词，忽略无关的词**。



### Transformer 的基础结构

不用记复杂细节，记住两大模块即可：

1. **编码器（Encoder）**：负责 “读懂输入”，把输入的文本转成语义向量，提取特征信息
   
   * 适合做理解类任务：分类、情感分析、语义检索

2. **解码器（Decoder）**：负责 “生成输出”，逐词预测下一个内容，生成完整回答
   
   * 适合做生成类任务：写文案、写代码、对话聊天

> 补充认知：我们常用的聊天大模型（比如 GPT 系列、豆包），都是「解码器 - only」架构，只保留了解码器部分，专门优化生成能力，这也是为什么它们生成内容能力强、对话流畅。



### 什么是知识蒸馏（Knowledge Distillation）

大白话定义：**让一个小模型（学生模型）去学习大模型（教师模型）的 “思考方式”，让小模型在体积小、速度快、成本低的前提下，获得接近大模型的效果**。

简单说就是：把大模型的能力 “蒸馏浓缩”，装进小模型里。

### 核心工作原理

蒸馏不是让小模型只背标准答案，而是学习大模型的**输出概率分布**。



### 什么是 Loop Engineering（循环工程）

大白话定义：**专门针对 Agent（智能体）的 “感知 - 规划 - 执行” 循环做优化，减少无效步骤、降低错误率、控制循环次数，让智能体用更少的调用次数、更低的成本，完成同样的任务**。



* 没优化的 Agent = 新手员工，做一步问一步，经常返工，绕弯路，一件事跑好多趟
* 优化后的 Agent = 资深员工，思路清晰，一步到位，很少返工，效率高
* Loop Engineering = 流程优化顾问，帮你梳理工作流程，去掉无用环节，增加校验节点，提升整体效率
